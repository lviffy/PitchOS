# Tether Developers Cup 🏆

> **Build on Tether. Play like a knockout tournament.**

The **Tether Developers Cup** is a global football-themed developer competition where teams compete in a knockout-style hackathon by building real-world applications using Tether's open-source developer stack.

Projects should be inspired by football and its surrounding ecosystem—fans, clubs, players, coaches, stadiums, tournaments, communities, fantasy leagues, prediction markets, ticketing, payments, analytics, AI coaching, and more.

Build something useful. Build something innovative. Surprise us.

---

# Competition Overview

Unlike traditional hackathons, the Developers Cup follows a tournament format.

You build **one project** and continuously improve it throughout the competition.

After each judging round, qualifying teams continue developing the same project until the next checkpoint.

# Build Tracks

Participants may build using **one or multiple** Tether technologies.

## Pears Stack (P2P)

Build decentralized peer-to-peer applications using the Pears ecosystem.

### Technologies

- Pear CLI
- pear-runtime
- Hyperswarm
- Hypercore
- Autobase

**Documentation**

https://docs.pears.com

---

## QVAC (Local AI)

Build AI applications that execute entirely on the user's device.

Examples include:

- Local LLMs
- AI assistants
- Speech processing
- Translation
- Search
- Offline AI

No cloud inference or external AI APIs are permitted.

**Documentation**

https://qvac.tether.io

---

## WDK (Wallet Development Kit)

Build self-custodial wallet and payment applications where users own their private keys.

Examples include:

- Wallets
- Payments
- Transaction tools
- Web3 authentication

**Documentation**

https://wdk.tether.io

# Competition Flow

The Developers Cup follows a progressive elimination format.

### Registration Phase

- Register before July 6.
- Start building immediately.

### Round One

- Submit your project and demo by July 8.
- Judges select the **Top 16**.

### Semifinals

- Continue improving your project.
- Submit again on July 12.
- Judges select the **Top 4**.

### Finals

- Finalists continue polishing their project.
- Final submission closes July 14.
- Live presentations take place between July 15–18.

### Winners

Champions are announced on **July 19**.

# Who Can Participate

The competition is open worldwide.

Requirements:

- 18 years or older
- Free to participate
- Any experience level
- Solo participants or teams of up to four members

Rules:

- One submission per participant
- Cannot participate both individually and in a team
- Cannot join multiple teams
- Every teammate must be listed on the DoraHacks project page

Tether employees and judges may mentor participants but are not eligible to win prizes.

---

# Project Requirements

Projects must make genuine use of the selected Tether technology.

Examples:

### QVAC

- All AI processing must execute locally using the QVAC SDK.
- Cloud AI services are not allowed.

### Pears

- Networking must use Pears technologies.
- Plain WebRTC alone is not sufficient.

Participants may use any programming language or framework.

All third-party services, APIs, and pre-built components must be disclosed.

---

# Submission Requirements

Every submission must include:

- Registration through DoraHacks
- Selected competition track(s)
- Public GitHub repository
- Open-source license (Apache 2.0 or MIT)
- Clear setup instructions
- YouTube demo video (maximum 3 minutes, Unlisted)
- Disclosure of any pre-existing code

Judges will evaluate only work completed during the official competition period.

---

# Judging Criteria

Projects are scored from **1–5** across five categories:

1. Technical Ambition
2. User Experience
3. Real-World Utility
4. Creativity
5. Meaningful Use of the Selected Track(s)

Judges may request a live demonstration or technical walkthrough.

All judging decisions are final.
