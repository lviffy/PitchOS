# Tether Developers Cup – Rules

## Overview

The **Tether Developers Cup** is a global knockout-style hackathon focused on building real-world applications using Tether's open-source developer stack.

By participating, you agree to the official **Terms of Participation** and all rules described below.

---

# Eligibility

- Anyone may participate provided they meet the eligibility requirements described in the Terms of Participation.
- Each participant may submit **only one project**.
- You may participate:
  - Individually, or
  - In a team of **up to 4 members**.
- You **cannot**:
  - Participate both solo and in a team.
  - Join multiple teams.
- Team members must all be listed on the project's **DoraHacks** page.

---

# Available Tracks

Projects may compete in one or more tracks.

## Pears

Peer-to-peer applications built using the **Pears / Holepunch stack** running on Bare.

- Website: https://pears.com

### Requirements

- Networking must use the Pears/Holepunch ecosystem.
- Examples include:
  - Hyperswarm
  - Bare runtime
- Pure WebRTC alone does **not** satisfy this requirement.

---

## QVAC

On-device AI applications powered by the **QVAC SDK**.

- Website: https://qvac.tether.io

### Requirements

All AI workloads must execute **locally on the user's device**, including:

- Inference
- Embeddings
- Retrieval-Augmented Generation (RAG)
- Multimodal processing
- Speech models

Cloud AI APIs are **not permitted**.

---

## WDK

Self-custodial wallet applications using the **Wallet Development Kit**.

- Website: https://wdk.tether.io

---

# Open Source Requirement

Every submission must:

- Be hosted on a public GitHub repository.
- Remain accessible to judges.
- Be released under the **Apache 2.0 License** during the hackathon and for a reasonable period afterward.

---

# Demo Video Requirements

Every submission must include:

- A YouTube video
- Visibility set to **Unlisted**
- Maximum length:
  - **3 minutes**

The video should clearly demonstrate:

- The problem
- The solution
- The application running
- Core features

### Additional Requirement

Starting from the **Round of 16**, submissions must also include a screen recording showing the application running.

---

# Project Requirements

Projects should be easy for judges to evaluate.

Each submission should include:

- Clear setup instructions
- Easy execution
- Browser access or runnable application
- Working build

---

# Technology Rules

Participants may use **any programming language or framework**.

However:

- The chosen Tether track must be integrated meaningfully.
- Setup and testing instructions must be provided.
- Any third-party APIs, services, SDKs, or components must be disclosed.

---

# Tournament Timeline

The Developers Cup follows a knockout tournament format.

| Stage | Date |
|---------|------|
| Registration Opens | June 28, 2026 |
| Registration Closes | July 6, 2026 |
| Building Begins | June 28, 2026 |
| Elimination Rounds | July 7–14, 2026 |
| Final Submission Deadline | **July 14, 2026 (23:59 UTC+1)** |
| Live Final | July 15, 2026 |

Participants continue improving **the same project** throughout every round.

Projects are **not restarted** after each elimination stage.

---

# Theme Requirement

Projects must relate to football and the global tournament ecosystem.

Examples include:

- Fans
- Teams
- Clubs
- Coaches
- Matches
- Predictions
- Watch parties
- Ticketing
- Fantasy experiences
- Communities
- Tipping

Projects outside the theme will not be judged.

# Submission Checklist

Each submission must include:

- Product name
- Short description
- Selected track(s)
- Represented nation
- Team members
- Team background
- Team location
- Public GitHub repository
- Description of how each selected Tether track is used
- Unlisted YouTube demo (≤ 3 minutes)

Participants are encouraged to allocate sufficient time to produce a clear and professional demo video.

---

# Judging Criteria

Projects are evaluated holistically.

Judges prioritize:

- Strong engineering
- Meaningful platform usage
- Working software
- Real-world usefulness

A polished UI is appreciated but does not outweigh technical quality.

---

## Pre-existing Code

Participants may:

- Use existing code
- Build on open-source software
- Use existing protocols

However:

- All prior work must be disclosed.
- Judging focuses on work completed during the official hackathon period.

---

## Scoring Categories

Each category is scored from **1–5**.

### 1. Technical Ambition

- Difficulty of the problem
- Engineering quality
- Technical execution

---

### 2. User Experience

- Ease of use
- Clarity
- Overall usability

---

### 3. Real-World Utility

- Practical value
- Likelihood of real adoption

---

### 4. Creativity

- Originality
- Innovation
- Freshness of the idea

---

### 5. Tether Platform Usage

Meaningful use of the selected platform:

- Pears
- QVAC
- WDK

Projects should leverage the platform for what it is designed to do—not merely include it superficially.

---

# Live Demonstration

Judges may request teams to:

- Run the project live
- Demonstrate functionality
- Walk through implementation details

Organizer and judge decisions are final.
